# 🍽️ نظام إدارة مطعم تفاعلي (CLI) | Restaurant Management System

> مشروع تطبيقي بلغة **Python**، مبني بأساسيات البرمجة الكائنية (OOP) بعد إتمام دورة **CS50P**. يحاكي نظام طلبات واقعي في مطعم، مع واجهة سطر أوامر نظيفة، وهيكلة برمجية قابلة للتوسع والصيانة.

---

## 📖 عن المشروع
يهدف هذا المشروع إلى تطبيق المفاهيم الأساسية للبرمجة الكائنية في سيناريو واقعي: **إدارة طاولات مطعم، تتبع الطلبات، حساب الفواتير، وتطبيق خصومات العملاء المميزين (VIP)**.  
تم تصميم النظام بعزل المنطق البرمجي عن واجهة المستخدم، مما يجعله أساساً متيناً يمكن تحويله لاحقاً إلى واجهة رسومية، تطبيق ويب، أو نظام متكامل مع قاعدة بيانات.

---

## ✨ المميزات الرئيسية
- ✅ واجهة تفاعلية بالكامل عبر سطر الأوامر (CLI)
-  طباعة فاتورة مفصلة عند إتمام الطلب
- 💎 دعم وضع `VIP` بخصم تلقائي 10%
-  إعادة تعيين حالة الطاولة تلقائياً بعد الدفع لاستقبال عميل جديد
- 🛡️ حماية من إدخال البيانات الخاطئة (`Defensive Programming`)
- 📦 هيكل معياري يفصل بين تعريف الأصناف، منطق الطاولات، وواجهة التفاعل

---

## 🛠️ المفاهيم التقنية المستخدمة
| المفهوم | التطبيق في الكود |
|:--|:--|
| **Encapsulation & Properties** | استخدام `@property` لحساب الإجمالي ديناميكياً مع خصم VIP، وإخفاء `_total` لمنع التعديل المباشر |
| **Composition over Inheritance** | `Hall` يحتوي على قائمة من `Table` بدلاً من التوريث غير الضروري |
| **Single Source of Truth** | قائمة `MENU_ITEMS` موحدة تغطي جميع الأصناف مع حقل `category` للتصنيف |
| **Type Hints & Documentation** | توثيق الأنواع (`-> str`, `-> float`, `-> None`) لتحسين القراءة ودعم أدوات التحليل الثابت |
| **Lifecycle Management** | دورة حياة واضحة: `init → add → checkout → reset` لضمان عزل الطلبات |
| **DRY Principle** | دالة موحدة للتحقق من المدخلات `get_int_input()` ومنع تكرار كود التحقق |
